## General steps

1) Create a new folder in `subworkflows/deconvolution`, containing:
   - A script which (minimally) takes as **input** a single-cell reference matrix, spatial expression matrix, and cell type annotation column,  
     and returns as **output** a TSV file of the predicted proportions (spots in rows and cell types in columns).
   - An entry in the Snakemake workflow (`Snakefile`) that calls this script.

2) Include the task in `subworkflows/deconvolution/rules.mk`

### Example: implementing NNLS in R

We will be adding a simple algorithm of non-negative least squares regression to the pipeline. For R methods, we assume the single-cell reference is a Seurat Object with a cell type annotation column, and the spatial object is either a Seurat object or a synthetic dataset generated by synthspot (a named list with the expression matrix in "counts").

1. Create a directory `subworkflows/deconvolution/nnls` containing:
   - `script_nf.R`: a script that runs NNLS. We use `R.utils::commandArgs` to parse command line arguments, with `sc_input` and `sp_input` as the path to the single-cell and spatial objects, and `annot` as the name of the cell type annotation column.
     <br>The script returns a TSV file of the spot x cell type proportion matrix. We recommend copying the template of results printing, 
     because we 1) do not include rownames, 2) remove non-alphanumeric characters from cell types and 3) shell sort the cell types.
   - **OPTIONAL:** build a docker container with your method (see `Dockerfile`)

2. Add nnls to `subworkflows/deconvolution/rules.mk`
   - Define a rule in `rules.mk`:
     ```make
     nnls_results: 
         Rscript scripts/nnls/script_nf.R $(sc_input) $(sp_input) $(annot)
     ```
   
3. Include the new task in the Snakemake workflow (`Snakefile`):
    ```python
    include: 'subworkflows/deconvolution/rules.mk'

    rule run_nns_methods:
        input:
            expand('nnls_results.tsv', sc_input=sc_inputs, sp_input=sp_inputs)
    
    rule all:
        input:
            run_nns_methods()
    ```

4. Test it out with
   ```
   snakemake --profile local \
             --config sc_input=unit-test/test_sc_data.rds sp_input=unit-test/test_sp_data.rds annot=subclass
   ```

### Extra information for Python methods

For simple algorithms like NNLS, the workflow is exactly the same but the inputs are expected to be h5ad files instead of Seurat objects. However, most Python methods make use of Bayesian probabilistic models (i.e., cell2location, stereoscope, and DestVI) and comprise model building and model fitting steps. Hence, you would need two scripts and two Snakemake rules.

Typically, the model building script (`build_model.py`) only takes the single-cell object and annotation column as input, while the model fitting script (`fit_model.py)` takes the model and spatial dataset as input. The Snakemake rules in `rules.mk` would minimally look something like:

```make
model:
    python scripts/build_model.py $(sc_input) --annot $(annot)

fit_model: model
    python scripts/fit_model.py $(sp_input) $<

output_props: fit_model
```

**Note:** In the cell2location/stereoscope/DestVI tasks, you will see input tuples like `$(sp_input), $(sp_input_rds)`. This is because although we internally converted the RDS file to a H5AD file, the original RDS file is still needed for metric computation. You will also need to follow this format while implementing your own method.

Then, you can also add the method in `subworkflow/deconvolution/rules.mk`:

- Define rules similar to those above.
- Include them in the main Snakemake workflow (`Snakefile`) with necessary dependencies.

Adding your process to the Snakemake workflow is slightly more complicated than the R case, since we want to be able to run multiple spatial datasets by building the model only once. Here's an example using `cell2location`:

```python
# Define rules for cell2location tasks in Snakefile

rule buildCell2locationModel:
    input: 
        sc_input=rules.sc_input.output
    output: 
        "model_file"
    shell:
        """
        python scripts/build_model.py {input.sc_input} --annot $(annot)
        """

rule fitCell2locationModel:
    input: 
        model="model_file", sp_input=sp_inputs_h5ad, sp_input_rds=sp_inputs_rds
    output:
        "output_props"
    shell:
        """
        python scripts/fit_model.py {input.sp_input} {input.model}
        """

rule formatC2L:
    input: 
        "fitCell2locationModel"
    output:
        "formatted_output.tsv"
    script:
        "scripts/format_c2l.sh"

# Ensure dependencies in Snakefile
include: 'subworkflows/deconvolution/rules.mk'

rule all:
    input: 
        formatted_output="formatted_output.tsv"
```

This setup provides a general framework for integrating deconvolution tools using Snakemake and Makefiles, allowing for modular and scalable workflow management. Adjust paths and names as necessary to fit your specific project structure.
